# Data Directory

This directory stores simulation output data files generated by the costly signaling ABM.

## Data Files

Data files are **not included** in this repository to keep the package size manageable. All data can be regenerated using the provided scripts.

### Expected Files After Generation

| File | Size | Generation Command | Time |
|------|------|-------------------|------|
| `temporal_validation_results_*.pkl` | ~500 KB | `make data-quick` | ~15 min |
| `parameter_sensitivity_results_*.pkl` | ~10 KB | `make data-quick` | ~15 min |
| `sensitivity_analysis_results_*.pkl` | ~800 MB | `make data-full` | ~24-48 hrs |

### File Format

All data files use Python's pickle format (`.pkl`) and can be loaded with:

```python
import pickle

with open('data/sensitivity_analysis_results_20260112.pkl', 'rb') as f:
    results = pickle.load(f)
```

## Data Generation

### Quick Validation Data (~30 minutes)

```bash
make data-quick
```

This generates the validation data used for Figures 9 and 10.

### Full Sensitivity Analysis (~24-48 hours)

```bash
make data-full
```

This runs the complete parameter space exploration:
- 208 parameter combinations
- 50 replicates per combination
- 10,400 total simulations

## Data Structure

### Sensitivity Analysis Results

```python
{
    'results': [
        {
            'frequency': float,           # Shortfall frequency (years)
            'magnitude': float,           # Shortfall magnitude (0-1)
            'monument_win_rate': float,   # Fraction of runs where monuments win
            'avg_final_population': float,
            'population_std': float,
            ...
        },
        ...
    ],
    'metadata': {
        'date': str,
        'n_replicates': int,
        'random_seed': int,
        ...
    }
}
```

### Temporal Validation Results

```python
{
    'condition': str,               # e.g., 'frequent_severe'
    'monument_populations': array,  # Time series for each replicate
    'reproduction_populations': array,
    'years': array,
    ...
}
```

## Reproducibility

All simulations use explicit random seeds for reproducibility:

- **Base seed**: `20260112` (date-based)
- **Per-replicate**: `base_seed + replicate * 1000 + condition_hash`

Due to floating-point differences across platforms, exact numerical values may vary slightly, but qualitative patterns should be identical.
